# PRD — 반도체 시료 생산주문관리 시스템 (SampleOrderSystem)

이 문서는 `[CRA_AI] Day3_개인과제_반도체시료관리_r1 2.pdf` 명세를 기준으로 정리한 요구사항 문서입니다. 코드 아키텍처/기술스택/개발 방법에 대한 안내는 `CLAUDE.md`를 참고하세요.

## 1. 배경

가상의 반도체 회사 "S-Semi"는 다양한 종류의 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 납품한다. 시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.

기존에는 엑셀과 메모장으로 주문을 관리하여 아래와 같은 문제가 있었다.

- 주문 처리 여부를 파악하기 어려움
- 공정 예약 후 완성 시점을 알기 어려움
- 재고가 충분한데도 불필요한 추가 공정이 도는 경우 발생

이를 해결하기 위해 "반도체 시료 생산주문관리 시스템"을 개발한다.

## 2. 역할 및 흐름

| 역할 | 설명 |
| --- | --- |
| 고객 | 필요한 시료를 이메일 등으로 요청(시료 요청자) |
| 주문 담당자 | 요청에 맞게 주문서를 작성/관리 (주문 접수, 승인/거절 결과 확인) |
| 생산 담당자 | 개발 시료 등록, 주문 수신 후 승인 또는 거절, 시료 생산 |

흐름: 고객 → (시료 요청) → 주문 담당자 → (주문서 전달) → 생산 담당자 → (승인/거절) → 주문 담당자

이 시스템은 위 역할들이 각자 콘솔에서 직접 명령을 입력해 시료를 등록하고 주문을 처리하는 **콘솔(CUI) 기반** 애플리케이션이다.

## 3. 시스템 개요

### 3.1 생산 라인

- 공장에서 시료 하나를 생산하는 설비 흐름.
- 하나의 생산 라인은 시료를 하나씩 생산한다(단일 라인, 동시에 1개 품목만 생산).
- 주문이 들어온 시료에 대해서만 생산한다(재고 예측 생산 없음).

### 3.2 주문 상태 흐름

모든 주문은 아래 상태 중 하나를 가진다.

| 상태 | 의미 |
| --- | --- |
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (정상 흐름 외의 상태이며 모니터링에서 제외) |
| PRODUCING | 주문 승인 완료 및 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료 및 출고 대기 중 |
| RELEASE | 출고 완료 |

상태 전이:

```
RESERVED --승인(재고충분)--> CONFIRMED --출고--> RELEASE
RESERVED --승인(재고부족)--> PRODUCING --생산완료--> CONFIRMED --출고--> RELEASE
RESERVED --거절--> REJECTED
```

## 4. 기능 명세

### 4.1 메인 메뉴

- 기능별 선택 화면을 표시한다.
- 전체 시료에 대한 요약 정보(등록 시료 종수, 총 재고, 전체 주문 건수, 생산라인 대기 등)를 확인할 수 있어야 한다.

| 메뉴            | 의미                             |
|---------------|--------------------------------|
| 시료 관리         | 새로운 시료 등록, 목록 조회, 이름 검색 기능     |
| 주문 (접수/승인/거절) | 고객 주문 접수 및 생산 라인 담당자의 승인·거절 처리 |
| 모니터링          | 상태별 주문 수 및 시료별 재고 현황 확인        |
| 출고 처리         | CONFIRMED 상태 주문에 대해 출고 실행      |
| 생산 라인 조회      | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인    |
| 종료            | 현재 프로그램을 종료                    |

화면 구성(레이아웃)은 자유롭게 결정한다 — PDF의 예시 UI는 참고용일 뿐 강제 사양이 아니다.

### 4.2 시료 관리

시료(Sample)는 이 시스템의 가장 기본이 되는 단위다. 각 시료는 고유한 이름과 속성을 가지며, **시스템에 등록된 시료만 주문 가능**하다.

| 메뉴    | 의미                                            |
|-------|-----------------------------------------------|
| 시료 등록 | 새로운 시료를 시스템에 추가. 속성 값: 시료 ID, 이름, 평균 생산시간, 수율 |
| 시료 조회 | 등록된 모든 시료 목록을 확인. 현재 재고 수량도 함께 표시             |
| 시료 검색 | 이름 등 속성으로 특정 시료를 검색                           |
| 뒤로    | 상위 메뉴로 이동                                     |

**수율(收率)**: (정상적인 시료 수 / 총 생산 시료 수). 예) 100개 생산 중 정상품 90개 = 0.9

시료 조회와 시료 검색 시에 나오는 결과 화면은, 시료 ID, 이름, 평균 생산시간, 수율, 현재 재고량 이 나와야 한다.

### 4.3 시료 주문

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다.

**시료 예약(Reserve)**

- 고객이 원하는 시료와 수량을 주문한다.
- 이 시점에서 주문 상태는 `RESERVED`.
- 예약 시 입력 값: 시료 ID, 고객명, 주문 수량.

### 4.4 주문 승인/거절

- 접수된 주문(`RESERVED`) 목록을 확인한다.
- 목록 화면에는 번호(1,2,3 순차 증가), 주문 번호, 고객, 이름, 수량, 상태 가 표기된다.
- 관리할 번호(1,2,3 순차 증가)를 입력받으면, 해당 시료에 대한 정보와, 부족분과 재고량이 표기되고, 부족한 경우 예상 소요시간을 계산한다. (이 때 소요시간은 생산 큐 순서에 맞춰 누적된 시간으로 보여준다.)
- 해당 주문에 대하여 승인 혹은 거절을 선택할 수 있게 한다.

**접수된 주문 목록**: `RESERVED` 상태의 주문 목록을 표시한다.

**주문 승인**: 접수된 특정 주문에 대해 승인한다. 승인 시 재고 상황에 따라 2가지 방식으로 자동 처리된다.

- 재고가 충분한 경우 → 주문을 즉시 `CONFIRMED` 상태로 전환.
- 재고가 부족한 경우 → 생산 라인에 자동으로 등록하고, 주문 상태를 `PRODUCING`으로 전환.

**주문 거절**: 접수된 특정 주문에 대해 거절한다. 즉시 `REJECTED` 상태로 전환한다.

### 4.5 모니터링

주문에 따른 시료 재고에 대한 정보 모니터링 기능이며, 담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록 구성한다.

**주문량 확인**

- 현재 상태별(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE`) 주문 건수를 확인한다.
- `REJECTED`는 유효한 주문이 아니므로 무시(집계 제외)한다.

**재고량 확인**

- 각 시료별 현재 재고 수량과 상태를 표기한다.
- 출력 순서는 재고 내림차순으로 보여준다.
- 상태는 총 3가지이다.
  - 여유: 주문 대비 재고 충분 상태
  - 부족: 주문 대비 재고 수량 부족 상태
  - 고갈: 수량이 0인 상태

**뒤로**

- 상위 메뉴로 이동한다.

### 4.6 생산라인

생산라인에 대한 정보를 표시한다. 주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려하여 시료를 생산한다.
상위 메뉴로 갈 수 있도록 "뒤로" 메뉴도 추가되어야 한다.

- 실 생산량 = `ceil(부족분 / 수율)`
- 총 생산 시간 = `평균 생산시간 * 실 생산량`
- 생산 완료 시 주문 상태 `PRODUCING -> CONFIRMED` 변경.

**생산 현황 표기**: 현재 생산 중인 시료에 대한 정보를 표기한다(주문 정보, 시료 이름, 주문량, 재고, 부족량, 실 생산량, 진행율과 남은 시간이 표기된다.

**대기 주문 확인**: 생산라인의 대기열(생산 큐)을 이용해 생산 작업을 대기하고 있는 목록을 출력한다. 생산 라인은 1개뿐이며, 스케줄링 전략은 **FIFO**다.
                  보여주는 정보는 FIFO 에 의한 순서, 주문번호, 시료 이름, 주문량, 부족분, 실생산량, 예상 완료시간이다.

### 4.7 출고 처리
상위 메뉴로 갈 수 있도록 "뒤로" 메뉴도 추가되어야 한다.

- 재고가 충분해진 `CONFIRMED` 주문에 대하서만 보여준다. (번호, 주문번호, 고객, 시료 이름, 수량 표기)
- 출고할 번호를 입력받으면 해당 주문 상태가 `RELEASE`로 전환되며, 처리 정보를 보여준다.
- 출고 수량만큼 해당 시료의 재고를 차감해야 한다.
